Work Strategy

  1. 1. Understanding the business need

    Understanding need is the first step where, we start from going through the application modules, flow, input/output those help to design the dynamic, scalable and secured architecture.

  2. 2. Crafting the Architecture

    The most important and crucial step that is covering to select the cloud service provider, finding the right services, resources and its type. We study requirement deeply. We go through each module, its processes and flow to learn...

    1. a). What cloud services will we use?

    2. b). How these services are going to integrated?

    3. c). What are the best resources and their types we needed?

    4. d). Where and what are security need to apply?

    5. e). Where the monitoring needed?

    To find out the answer of above, we study the technologies, best practices, white-papers, etc. With selecting right resources we also more focus on the security and monitoring part.

  3. 3. Implementing Architecture

    We implement architecture with the best practices. Into the distributed architecture, multiple cloud services are together executing, so it need to test complete flow of each process to identify its proposed execution. With setting up the ideal monitoring, we perform the security checks to ensure.

  4. 4. Monitoring

    Each application, must have monitoring not only on security prospect but also on performance and execution view. Monitoring help us to know…

    1. a. Where and what update we need?

    2. b. Where the issue of execution is happening?

    Other major help, that monitoring allow is the measuring the resource sizing that help a to have right sizing on right resources always save the cost. Having precise logging into the application can help track the threats easily.

  5. 5. Keep improving

    Monitoring help to identify the needed improvement of the resources time to time in terms of resource sizing, security patches, version updating, etc. Also improvement needed when requirement updates.
